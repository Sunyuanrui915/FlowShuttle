# 流梭 Flow Shuttle 测试数据目录使用说明

流梭不在应用界面中提供“生成测试数据 / 清空测试数据”入口。开发和回归验证时，请使用独立测试数据目录，避免污染真实用户数据。

## 生成测试数据

```bash
npm run create:test-data
```

默认输出目录：

```text
dev-data/flow-shuttle-test-data/
```

目录中会生成：

- `flow-shuttle.sqlite`
- `README.md`

如果目录中已经存在测试数据库，脚本会拒绝覆盖。需要重新生成时可显式执行：

```bash
npm run create:test-data -- --force
```

`--force` 只会替换 `dev-data/flow-shuttle-test-data/` 中的测试数据库文件，不会触碰 Electron `userData` 或你的真实数据目录。

## 在应用中使用测试数据

1. 运行脚本生成测试数据目录。
2. 打开流梭。
3. 进入 Settings。
4. 点击“使用已有数据目录”。如果你使用的是当前支持切换已有数据库的“更改数据目录”入口，也可以选择同一个目录并确认切换。
5. 选择 `dev-data/flow-shuttle-test-data/`。
6. 应用会读取该目录中的测试项目、工作项、日报、周报、月报和热力图数据。
7. 测试完成后，再通过 Settings 切回你的真实数据目录。

## 数据覆盖范围

测试数据库包含：

- `[测试] 流梭`
- `[测试] 个人网站`
- `[测试] 数据治理报表系统`
- `[测试] 公众号内容运营`

覆盖场景包括：

- 至少 3 周记录
- 当前月份数据
- 今天和上一工作日
- 周末无记录
- 轻度记录日、深度单项记录日、多项浅记录日
- 长文本、多段 Markdown、下一步计划、阻碍 / 风险
- 今日完成、暂停、搜索、删除验证入口
- 日报、周报、月报、热力图和 AI 提炼版示例
