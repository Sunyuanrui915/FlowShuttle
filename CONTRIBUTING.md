# Contributing To Flow Shuttle

Flow Shuttle is currently maintained primarily by the main maintainer. Feedback, bug reports, and focused pull requests are welcome.

## Issues

- Use Issues to report bugs, usability problems, and documentation gaps.
- For larger features, please open an Issue first so the scope can be discussed before implementation.
- Include reproduction steps, expected behavior, actual behavior, and screenshots when useful.

## Pull Requests

- Bug fixes and small documentation improvements can be submitted directly as pull requests.
- Larger product or architecture changes should start with an Issue.
- Keep changes incremental and focused.
- Avoid unrelated refactors.
- Do not introduce new dependencies unless necessary and explained.

## AI-Assisted Contributions

AI-generated code is allowed, but contributors must review it before submission. Pull requests should clearly describe:

- What changed.
- Why it changed.
- How it was tested.
- Any limitations or follow-up work.
